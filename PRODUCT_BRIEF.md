# Product Brief: Scriptorium

## Purpose

Scriptorium is a custom research platform for serious theological and historical study. It is designed for reading, marking up, interrogating, citing, and storing scholarly texts and the research artefacts generated from them.

## Intended user

The initial user is a theologian and church historian preparing for doctoral work. The product should therefore privilege academic workflows over casual reading: durable notes, source provenance, citation accuracy, page-number fidelity, retrieval, and synthesis.

## Core problem

Existing tools split the research act across too many systems: PDF readers, citation managers, cloud drives, note apps, word processors, and AI chat interfaces. Scriptorium should unify the workflow without flattening the distinction between source text, user annotation, bibliographic metadata, and later interpretation.

## Scholarly object

The primary scholarly object is:

> source text + location + book page system + annotation + citation + research question + later retrieval

The app should preserve the original file and store scholarship as structured overlay data.

## Required capabilities

- Display PDF files.
- Display or ingest Microsoft Office files, especially Word documents.
- Display or ingest Google Workspace documents.
- Display plain text and Markdown files.
- Support ten highlight colors.
- Preserve book page numbering, not merely PDF page indexes.
- Store highlights, notes, tags, page references, and citation metadata.
- Generate citations on the fly using major academic styles.
- Prioritize citation systems used in theology, biblical studies, religious studies, history, and adjacent humanities disciplines.

## Citation priorities

The first citation styles should include:

- Chicago notes and bibliography
- Turabian notes and bibliography
- SBL Handbook of Style
- APA
- MLA
- Harvard-style author-date

## First product milestone

A user can upload/register a PDF, map book page numbers, highlight a passage, add a note, generate a Chicago or SBL footnote using the correct page locator, save the result, and recover it after reload.
